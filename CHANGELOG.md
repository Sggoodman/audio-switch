# Changelog

All notable changes to this project will be documented in this file.

## [v1.0.3] - 2026-05-20

### Added
- 快捷键录制功能：点击"录制快捷键"按钮，按下组合键即可设置，支持 Esc 取消
- 设备热插拔检测：USB 耳机等设备拔插后托盘菜单自动刷新
- "关于"对话框：托盘菜单新增关于入口，显示版本号
- 设备类型图标：托盘菜单根据设备类型（音箱/耳机/HDMI）显示不同图标
- 日志轮转：使用 lumberjack 自动管理日志文件（单文件 5MB，保留 3 份备份）
- 共享工具函数：提取 `internal/util.GetExePath()` 消除重复代码

### Fixed
- 快捷切换设备失败：蓝牙等设备重连后 ID 变化导致切换失败，新增名称回退匹配
- 设备下拉选择不保存：配置中的设备名不在活跃列表时，首次选择被跳过
- 设置快捷键时误触发切换：录制前先抑制热键回调，防止竞态条件
- saveConfig 误报成功：错误分支缺少 return，修复后仅成功时记录日志
- GetConfigPath 覆盖测试路径：sync.Once 内增加已设置路径检查
- COM 线程安全：所有 Windows Audio API 调用添加 LockOSThread

### Changed
- 重构设置窗口：提取 buildDeviceSelect/buildVolumeSlider 等方法，消除重复代码
- 滑块防抖：音量滑块拖动时 300ms 防抖写盘，避免频繁 IO
- 统一日志系统：全面迁移至 zap 结构化日志，按模块分类输出
- 设置窗口复用：重复打开不再创建新窗口，刷新内容即可
- 构建脚本：版本号改为 git describe 自动获取，注入到关于对话框
- 统一通知+刷新：提取 notifyAndRefresh 方法，消除 switchDevice/QuickSwitch 重复逻辑

---

## [v1.1.0] - 2026-05-17

### Added
- Windows 系统通知：切换设备时弹出 Toast 通知，可在设置中开关
- 开机自启动：设置界面支持启用/禁用开机自动启动

### Fixed
- 音量持久化修复：保存和加载配置时正确处理音量值
- 默认音量调整为 40%

### Changed
- 优化通知系统并改进构建流程
- 隐藏终端窗口，GUI 模式运行

---

## [v1.0.0] - 2026-05-16

### Added
- 跨平台音频输出设备快速切换工具
- 系统托盘菜单，列出所有活跃音频输出设备
- 快捷切换（A/B）：在两个偏好设备之间一键切换
- 音量预设：为每个偏好设备设置切换后的目标音量
- 全局热键：默认 Ctrl+Alt+S 快速切换，可自定义
- 偏好设置窗口：设备选择、音量调节、热键配置、通知开关
- 纯 Go 实现 Windows 音频切换，无外部依赖

### Changed
- 从 uTools 插件重构为独立桌面应用
- 使用 Windows Core Audio COM API 替代 Node.js/koffi 方案
